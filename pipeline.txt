### 6 luồng thu thập (thực tế đang chạy)

#### Luồng 1 — Nến 1m (snapshot mỗi 1s)
```
Stream:   wss://fstream.binance.com/market/ws/btcusdt@kline_1m
Ghi:      mỗi 1 giây (snapshot giá tại thời điểm đó)
File:     klines_1s.csv
Columns:  open_time, open, high, low, close, volume,
          taker_buy_vol, num_trades

Lưu ý: Binance Futures không có kline_1s nên dùng kline_1m
        update 250ms, lấy snapshot mỗi 1s → đủ độ phân giải
```

#### Luồng 2 — Liquidation realtime
```
Stream:   wss://fstream.binance.com/market/ws/!forceOrder@arr
Ghi:      ngay lập tức khi có event
File:     liquidations.csv
Columns:  event_time, symbol, side, price, qty, usd_value

side = BUY  → lệnh SHORT bị liquidate (sàn buộc mua → giá bị đẩy lên)
side = SELL → lệnh LONG bị liquidate  (sàn buộc bán → giá bị đẩy xuống)
```

#### Luồng 3 — Order Book Depth (MỚI — quan trọng nhất)
```
Stream:   wss://fstream.binance.com/stream?streams=btcusdt@depth20@500ms
          ⚠️ Phải dùng /stream?streams= (KHÔNG phải /market/ws/)
          Binance wrap data trong {"stream": "...", "data": {...}}

Ghi:      mỗi 1 giây (snapshot mới nhất trong 1s)
File:     orderbook.csv
Columns:  timestamp,
          bid1_price, bid1_qty, ..., bid5_price, bid5_qty,
          ask1_price, ask1_qty, ..., ask5_price, ask5_qty,
          mid_price, spread, bid_vol_total, ask_vol_total, imbalance

Features tính sẵn:
  mid_price     = (bid1 + ask1) / 2
  spread        = ask1 - bid1
  bid_vol_total = tổng qty 5 bid levels
  ask_vol_total = tổng qty 5 ask levels
  imbalance     = bid_vol / (bid_vol + ask_vol)
                  > 0.55 → áp lực mua
                  < 0.45 → áp lực bán
```

#### Luồng 4 — AggTrade + CVD (MỚI)
```
Stream:   wss://fstream.binance.com/market/ws/btcusdt@aggTrade
Ghi:      2 chế độ:
          - Lệnh ≥ $50K: ghi ngay lập tức
          - Lệnh < $50K: gom 1 giây, ghi 1 dòng tổng hợp

File:     aggtrades.csv
Columns:  timestamp, agg_id, price, qty, usd_value,
          is_buyer_maker, cvd_delta

is_buyer_maker = True  → bán chủ động (seller là taker)
is_buyer_maker = False → mua chủ động (buyer là taker)
cvd_delta = +qty nếu mua CĐ, -qty nếu bán CĐ
agg_id = "BATCH" cho dòng tổng hợp 1s

CVD (Cumulative Volume Delta):
  CVD tăng liên tục → phe mua đang tấn công
  CVD giảm liên tục → phe bán đang tấn công
  CVD diverge với giá → tín hiệu đảo chiều
```

#### Luồng 5 — Open Interest mỗi 30s
```
Endpoint: GET https://fapi.binance.com/fapi/v1/openInterest
File:     open_interest.csv
Columns:  timestamp, oi_btc, oi_usd

OI tăng → tiền mới vào thị trường → sắp có biến động
OI giảm → lệnh đang đóng bớt → thị trường hạ nhiệt
```

#### Luồng 6 — Funding Rate mỗi 1h
```
Endpoint: GET https://fapi.binance.com/fapi/v1/premiumIndex
File:     funding_rate.csv
Columns:  timestamp, funding_rate, next_funding_time

> +0.01% → thị trường nghiêng LONG quá nhiều
< -0.01% → thị trường nghiêng SHORT quá nhiều
```

### Các features cần tính

#### Nhóm 1 — Giá (từ klines_1s.csv)
```python
# Aggregate 300 dòng gần nhất (5 phút × 60s)
price_change_5m   = (close_now - close_5m_ago) / close_5m_ago
price_change_1m   = (close_now - close_1m_ago) / close_1m_ago
volatility_5m     = std(close trong 5 phút)
volume_5m         = sum(volume trong 5 phút)
taker_buy_ratio   = sum(taker_buy_vol) / sum(volume)
# > 0.5 → áp lực mua đang mạnh hơn bán
```

#### Nhóm 2 — Liquidation (từ liquidations.csv)
```python
# Cửa sổ 5 phút gần nhất
liq_long_usd_5m    = sum(usd_value WHERE side='SELL')   # LONG bị quét
liq_short_usd_5m   = sum(usd_value WHERE side='BUY')    # SHORT bị quét
liq_total_5m       = liq_long_usd_5m + liq_short_usd_5m
liq_ratio_5m       = liq_short_usd_5m / (liq_total_5m + 1)
# > 0.7 → SHORT đang bị quét nhiều hơn

# Ước tính vùng thanh khoản từ lịch sử 4h
liq_zone_upper = percentile_90(price WHERE side='BUY',  last 4h)
liq_zone_lower = percentile_10(price WHERE side='SELL', last 4h)

dist_to_upper = (liq_zone_upper - current_price) / current_price
dist_to_lower = (current_price - liq_zone_lower) / current_price
# < 0.5% → giá rất gần vùng thanh khoản → dễ bị kéo vào
```

#### Nhóm 3 — Order Book (từ orderbook.csv — MỚI)
```python
# Lấy snapshot mới nhất
imbalance_now      = imbalance (đã tính sẵn trong CSV)
spread_now         = spread
mid_price_now      = mid_price

# Tính trung bình 60s gần nhất
imbalance_avg_1m   = mean(imbalance, last 60 rows)
imbalance_trend    = imbalance_now - imbalance_avg_1m
# > 0 → áp lực mua đang tăng
# < 0 → áp lực bán đang tăng

# Phát hiện "wall" lớn (bid hoặc ask volume đột biến)
bid_wall           = max(bid1_qty..bid5_qty)
ask_wall           = max(ask1_qty..ask5_qty)
wall_ratio         = bid_wall / (bid_wall + ask_wall + 1)
```

#### Nhóm 4 — CVD (từ aggtrades.csv — MỚI)
```python
# Tính CVD delta trong 5 phút gần nhất
# (chỉ dùng các dòng BATCH, bỏ qua large trade riêng lẻ)
cvd_delta_5m       = sum(cvd_delta WHERE agg_id='BATCH', last 5m)
# > 0 → phe mua đang kiểm soát
# < 0 → phe bán đang kiểm soát

# Đếm số lệnh whale trong 5 phút
whale_buy_count    = count(WHERE is_buyer_maker=False AND usd_value >= 50000)
whale_sell_count   = count(WHERE is_buyer_maker=True  AND usd_value >= 50000)
whale_net          = whale_buy_count - whale_sell_count

# Tổng USD whale
whale_buy_usd_5m   = sum(usd_value WHERE mua CĐ AND ≥$50K, last 5m)
whale_sell_usd_5m  = sum(usd_value WHERE bán CĐ AND ≥$50K, last 5m)
```

#### Nhóm 5 — Open Interest (từ open_interest.csv)
```python
oi_now             = oi_btc mới nhất
oi_30m_ago         = oi_btc 30 phút trước
delta_oi_30m       = (oi_now - oi_30m_ago) / oi_30m_ago

oi_5m_ago          = oi_btc 5 phút trước
delta_oi_5m        = (oi_now - oi_5m_ago) / oi_5m_ago
# delta tăng đột biến → tiền mới vào → sắp có biến động lớn
```

#### Nhóm 6 — Funding Rate (từ funding_rate.csv)
```python
funding_rate_now   = funding rate mới nhất (cập nhật mỗi 1h)
# > +0.01% → thị trường nghiêng LONG → dễ bị quét lên
# < -0.01% → thị trường nghiêng SHORT → dễ bị quét xuống
```

## TẦNG 3 — XÂY DỰNG LABEL

### Logic (không đổi, chỉ thêm context)
```
Tại thời điểm T:
  - features đã tính xong
  - liq_zone_upper đã biết

Đợi 30 phút → nhìn lại klines_1s.csv:
  label = 1  nếu max(high) trong [T, T+30m] >= liq_zone_upper
  label = 0  nếu không
```

### Cron job
```python
# Chạy mỗi 5 phút
# Chỉ xử lý row có timestamp <= now - 30 phút và label == ''

def build_labels():
    df_features = pd.read_csv('features_5m.csv')
    df_klines   = pd.read_csv('data/klines_1s.csv', parse_dates=['open_time'])

    for idx, row in df_features[df_features['label'] == ''].iterrows():
        t_start = row['timestamp']
        t_end   = t_start + timedelta(minutes=30)

        if t_end > datetime.now(UTC):
            continue  # chưa đủ 30 phút

        mask     = (df_klines['open_time'] >= t_start) & (df_klines['open_time'] <= t_end)
        max_high = df_klines[mask]['high'].max()

        label = 1 if max_high >= row['liq_zone_upper'] else 0
        df_features.at[idx, 'label'] = label

    df_features.to_csv('features_5m.csv', index=False)
```

### Luồng mỗi 5 phút

```

2. Nếu model chưa train → in count labeled rows, bỏ qua predict
3. predict_signal() → nếu prob >= 0.70 VÀ R:R >= 1.5
       → ghi paper_trades.csv
       → gửi Telegram
4. check_outcomes() → điền WIN/LOSS/EXPIRED cho trade ≥ 30 phút
5. Mỗi 1 giờ: in thống kê win rate, total PnL
```

### Paper trades outcome logic

```
Duyệt klines_1s.csv trong [opened_at, opened_at+30m]:
  if high >= tp  → WIN   (hit TP trước)
  if low  <= sl  → LOSS  (hit SL trước)
  hết 30 phút   → EXPIRED (đóng ở giá cuối)
```

### Output file

```

  opened_at, signal, prob, entry, tp, sl, rr,
  closed_at, outcome, pnl_pct, hit_tp, hit_sl
```

### BE — Luồng hoạt động

#### broadcaster.py — background thread (push 1s)

```
DashboardConfig.ready() → broadcaster.start() → thread daemon

Mỗi 1 giây:
  1. _tail_lines(klines_1s.csv) → price, ts
  2. pd.read_csv(features_5m.csv).iloc[-1] → prob, liq_zones, imbalance, cvd, oi, funding
  3. pd.read_csv(paper_trades.csv) → signal active (outcome='', mở trong 30p)
  4. async_to_sync(channel_layer.group_send)("tick", tick_dict)
     → TickConsumer.tick_message() → ws.send(JSON)

Mỗi 60s: reload xgb_model.json nếu file thay đổi → tính prob real-time
```

#### REST views

```
GET /api/klines/?hours=2
  → load_klines_chart(): aggregate klines_1s.csv → 1m candles
  → [{ts, open, high, low, close, volume}, ...]

GET /api/signal/
  → load_signal_state(): feature row mới nhất + active signal
  → {current_price, liq_upper, liq_lower, dist_upper_pct,
     imbalance, cvd_5m, funding_rate, delta_oi_5m,
     feature_ts, active_signal}

GET /api/trades/?limit=30
  → load_trades(): 30 paper trades gần nhất
  → [{opened_at, prob, entry, tp, sl, rr, outcome, pnl_pct}, ...]

GET /api/liq/?hours=4
  → load_liquidations(): BTCUSDT liqs trong 4h
  → [{ts, side, price, usd_value}, ...]
```
